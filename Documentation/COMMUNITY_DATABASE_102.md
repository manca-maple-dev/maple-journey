## How Maple Uses This Data

### 1. **Keyword-Based Search**
When users ask: "Where can I find legal help?" or "I need a job"
- Searches communities by keywords: "legal", "job", "health", etc.
- Filters by user's province and city
- Returns 3-5 top-rated matches

### 2. **Direct Contact Information**
Maple responds with:
```
📍 Organization Name
   Address: [Full street address, city, province]
   ☎️ Phone: [(555) 123-4567]
   🕐 Hours: Mon-Fri 9am-5pm
   🗣️ Languages: English, French, Spanish, Mandarin
   ✅ VERIFIED | ⭐ 4.8/5 Rating
   📝 Specialization: Immigration law, work permits, refugee claims
```

### 3. **Context Injection**
- On every user message, Maple scans for resource/help keywords
- If found, queries communities database
- Injects formatted community data into system prompt
- LLM generates response with specific contact information

### 4. **Smart Matching**
- Considers user's profile (province, city, permit type, languages)
- Sorts results by: verified status → rating → relevance
- Avoids duplicates and irrelevant results
- Personalizes recommendations

## Implementation Details

### Database Schema (MongoDB)
```javascript
{
  name: String,
  address: String,
  city: String,
  province: String,
  phone: String,
  email: String,
  website: String,
  hours: String,
  specialization: String,
  services: [String],
  verified: Boolean,
  rating: Number,
  languages: [String],
  tags: [String]  // Optional: for additional indexing
}
```

### Indexes Created
- Text index on: name, specialization, services (for keyword search)
- Composite index on: province, city (for location filtering)
- Sorted by: verified (descending), rating (descending)

### Search Query Example
```python
communities = await db.communities.find({
    "$text": {"$search": keyword},
    "province": user_province,
    "city": {"$regex": user_city, "$options": "i"}
}).sort([("verified", -1), ("rating", -1)]).limit(5)
```
